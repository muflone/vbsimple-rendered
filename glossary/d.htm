  • Daemon
    Programma che opera in background in un server, in attesa di richieste dei computer cui esso si collegano.

  • DAO
    Acronimo di Data Access Object.

  • Data Access Object (DAO)
    Una delle tre interfacce più diffuse per l'accesso a fonti dati (database). È l'interfaccia di default per accedere a fonti dati tramite l'utilizzo del controllo Data ma rappresenta una tecnologia un po' antiquata.
    Vedi anche ActiveX Data Objects (ADO).

  • Database
    Letteralmente Base di dati. Consiste in un archivio dati di varia natura con la possibilità di inclusione di funzionalità atte alla manutenzione dei suoi dati (Query).
    Il modello di database più conosciuto è il modello relazionale in cui i dati sono organizzati in forma tabellare e le varie tabelle sono connesse tra loro tramite relazioni ed indici.
    Vedi anche Recordset, Provider ed Interfaccia.

  • Data Definition Language (DDL)
    Linguaggio di definizione dei dati ed indica quei linguaggi la cui funzione è quella di definire le strutture logiche dei dati, la suddivisione in sottostrutture e la creazione di regole o di Query.
    Il linguaggio di definizione dei dati più utilizzato è SQL.
    Vedi anche Data Manipulation Language (DML).

  • Data Manipulation Language (DML)
    Linguaggio di manipolazione dei dati ed indica tutti quei linguaggi atti ad estrarre o modificare i dati di una fonte dati (Database).
    Il linguaggio di manipolazione di fonti dati più conosciuto è SQL; spesso le istruzioni di un linguaggio DML consentono la creazione di Query e l'estrazione, quindi, di Recordset.
    Vedi anche Data Definition Language (DDL) e SQL.

  • DC
    Acronimo di Device Context.

  • DDE
    Acronimo di Dynamic Data Exchange.

  • DDL
    Acronimo di Data Definition Language.

  • Deallocare
    Liberare un'area di memoria precedentemente allocata per essere utilizzata.
    La deallocazione comporta la restituzione dell'area di memoria come zona libera, rendendola nuovamente disponibile agli altri programmi.

  • Debug (De-Bug)
    Togliere le cimici, eliminare cioè gli errori all'interno di un programma, prima di distribuirlo agli utilizzatori.
    A volte, dopo una fase di debug, successiva alla distribuzione del programma, viene pubblicato un'aggiornamento del programma con i bugs corretti, altre volte viene pubblicato un Service Pack che corregge gli errori.

  • Decimale
    Sistema di numerazione a base 10. Le sue cifre vanno dallo 0 al 9.
    È il sistema di numerazione utilizzato in Italia.
    Vedi anche Sistemi di numerazione.

  • Design Time
    Vedi Fase di Progettazione.

  • Device Context
    Puntatore ad una struttura dati contenente informazioni e funzioni su un dispositivo di output quali il video o una stampante. Viene utilizzato per indirizzare le operazioni di disegno su un particolare dispositivo ed acquisirne informazioni quali le dimensioni o la posizione.

  • DispID
    Abbreviazione di Dispatch ID ed indica un numero identificativo (ID) di una funzione nell'architettura Component Object Model (COM).
    Viene utilizzato internamente dal sistema operativo per identificare univocamente una funzione all'interno di una Classe ActiveX.

  • DLL
    Vedi Libreria.

  • DML
    Acronimo di Data Manipulation Language.

  • DNS
    Acronimo di Domain Name System, un sistema che consente di associare ad ogni indirizzo IP un nome di dominio composto da lettere.
    Vedi anche IP.

  • Download
    Letteralmente "scaricamento" ed indica il processo di trasferimento dati da un computer remoto verso il computer locale.
    È opposto a Upload.

  • Drag'n'drop
    Vedi Trascinamento.

  • DSN (Data Source Name)
    Letteralmente nome di origine dati. Consiste in un collegamento ad un database dislocato all'interno di una rete. La loro utilità si rileva in sistemi operativi protetti oppure in ambito Internet, dove i visitatori non hanno pieno accesso al disco.
    L'impostazione di un DSN permette l'accessibilità ad un database a determinati programmi, celando la posizione dove in realtà essi si trovano.
    Vedi anche ODBC.

  • Dynamic Data Echange (DDE)
    Sistema di comunicazione tra processi. Mediante l'utilizzo di un'area di memoria condivisa due applicazioni - client e server - possono scambiarsi dati sia in invio che in ricezione. Il processo di comunicazione tra le applicazioni è detto comunicazione DDE.
